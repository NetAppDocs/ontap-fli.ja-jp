= ONTAP FLI移行について学ぶ
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
このセクションでは、Foreign LUN Import（FLI）のインポートワークフローについて説明します。外部LUNの検出からインポートの実行、監視、最終的なクリーンアップまでのライフサイクル全体を網羅しています。ONTAPが確立されたバックエンド接続（FCまたはiSCSI）を使用して、ホストからプロビジョニング解除され、ONTAPイニシエータ専用にマッピングされた外部LUNを安全に検出し、二重提示のリスクがないことを保証する方法について説明します。ワークフローでは、外部ディスクがONTAP内でどのように識別され、マークされるか、宛先LUNがソースジオメトリに合わせてどのようにプロビジョニングされるか、およびソースデバイスと宛先デバイスをペアリングするために永続的なインポート関係がどのように作成されるかについて詳細に説明します。

このセクションでは、LUNの状態とタイミングの観点からオフラインFLIとオンラインFLIの運用上の違いを明確にするとともに、基盤となるONTAPインポートコマンドは両方のモードで同じであることを強調しています。インポートの開始、監視、スロットリング（一時停止/再開）、および完了の方法、ならびに必要に応じてオプションの検証を使用してデータ整合性を検証する方法について説明します。最後に、データコピーが完了した後にインポート関係を安全に削除し、デスティネーションLUNを完全な本番環境に移行してFLIワークフローを完了する手順を概説します。

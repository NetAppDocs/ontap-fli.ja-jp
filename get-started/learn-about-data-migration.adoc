= ONTAPデータ移行プログラムについて学ぶ
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
データ移行プログラムは、お客様が外部（サードパーティ）アレイから ONTAP へデータを移行し、NetApp 7-Mode システムから ONTAP へ LUN を移行するための実績のあるソリューションを提供します。Foreign LUN Import（FLI）は、この移行ポートフォリオの主要コンポーネントとして機能します。移行の成功率を向上させ、作業負荷を軽減するために、このプログラムは、効率的かつ一貫性のある移行を実行するために必要なツール、ガイダンス、およびイネーブルメントを提供し、お客様が NetApp テクノロジをより迅速かつ高い信頼性で導入できるよう支援します。

データ移行には、長時間のシステム停止、本番環境への負荷リスク、限られたリソース、専門知識の不足など、実際の運用上の課題が伴うことが多い。可用性に対する期待が高まり続け、ダウンタイムが許容されなくなるにつれ、移行は事業継続性を考慮して計画されなければならない。パフォーマンスへの影響、データの整合性、および潜在的なデータ損失は、あらゆる移行において共通の懸念事項であるため、FLIの中核となる概念とワークフローを事前に理解しておくことは、実行を効率化し、初期設定と運用上のオーバーヘッドを削減するのに役立つ。

FLIは、外部（サードパーティ）アレイLUNからネイティブONTAP LUNにユーザーデータをコピーするONTAPデータ移行機能です。ONTAPは、SCSIイニシエータとして動作することで移行を実行します。外部LUNからブロックを読み取り、宛先のONTAP LUNに書き込みます。

ONTAP FLIは、外部LUNにファイバチャネル（FC）経由でアクセスする場合にサポートされました。ONTAP 9.19.1以降では、バックエンドでiSCSIトランスポートを使用して外部アレイLUNにアクセスするためのiSCSI FLIサポートが追加されました。iSCSI FLIでは、ソースアレイへのバックエンドパスはiSCSI/TCPですが、ONTAP LUNへのフロントエンドホストアクセスはFCまたはiSCSIのいずれかです(バックエンドとフロントエンドは異なる場合があります)。

iSCSIとFC ONTAPのFLIは、オフラインインポート（インポート中にホストのI/Oが発生しない）とオンラインインポート（ホストは接続されたままで、切り替え時に短時間の停止が発生する）の両方をサポートしています。



[[key-terms]]
== 主要な用語

外部アレイ:: ONTAPを実行していないストレージシステム（7モードからONTAPへの移行コンテキストにおけるNetApp 7モードを含む）。
外部LUN:: 外部アレイ上のユーザーデータを、そのアレイのネイティブフォーマットで格納したアレイLUN。
インポート関係:: データコピーに使用される外部LUN（ソース）とONTAP LUN（宛先）間の永続的なペアリング。
LUNインポート:: 外部LUNからONTAP LUNにブロックを転送するプロセス。




[[high-level-methodology]]
== 高レベルの方法論

調査フェーズ:: ホスト、ストレージ システム、SAN/ネットワーク ファブリックなど、環境全体の完全なインベントリを作成し、何が存在し、すべてがどのように接続されているかを把握してください。
分析フェーズ:: 収集したインベントリを確認し、互換性、リスク、および運用上の制約に基づいて、各ホストおよびソースアレイに最適な移行方法を決定する。
計画フェーズ:: 移行計画の作成と検証、移行先のONTAPストレージのプロビジョニング、必要な移行・検証ツールと運用手順書の準備を行います。
実行フェーズ:: 選択したワークフロー（オンラインまたはオフライン）に従って、移行（インポート/コピー）を実行し、必要なホストの修復および切り替え作業を完了します。
検証フェーズ:: 移行されたデータと構成が期待どおりに動作することを確認し、ホストアクセスとマルチパスを検証し、運用引き継ぎのために最終的な状態と結果を文書化します。




[[intended-audience]]
== 対象読者

この情報は、ONTAPへのSAN（ブロック/LUN）データ移行を計画または実行している顧客、パートナー、およびNetAppフィールドチームを対象としており、特に外部LUNインポート（FLI）および関連する移行オプションを使用する移行を対象としています。

.主な読者
* 移行アプローチの計画、ONTAPの準備、変更期間の調整、ホストアクセス/マルチパスの検証を担当する顧客ストレージ管理者/SAN管理者。
* FLI、アプライアンスベースのオプション、またはホスト/アプリケーションベースの移行方法の中から選択し、期間と労力を見積もるための概要ガイダンスを必要とする移行リーダー/技術プロジェクトオーナー。
* NetAppおよびパートナーの専門サービスエンジニアによる、移行の範囲設定、計画策定、資格認定、および顧客支援のサポート（最も単純な移行を除くほとんどの移行に推奨）。


.読者は以下の事項について基本的な知識を持っている必要があります：
* SANの概念（LUNマスキング/マッピング、ゾーニングまたはiSCSIネットワーク、マルチパス/ALUA）、および標準的な変更管理手順。
* ONTAP管理の基本事項と、支援的な計画ツール（環境データ収集や相互運用性検証ユーティリティなど）の使用方法。

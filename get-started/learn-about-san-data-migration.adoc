= ONTAPへのSANデータ移行について学ぶ
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
SANデータの移行は、多くの場合、限られたダウンタイム期間、本番環境におけるリスク、および専門リソースの不足といった制約を受けます。ONTAP Foreign LUN Import（FLI）は、外部（サードパーティ）アレイLUNからネイティブONTAP LUNへデータをコピーする組み込みのブロック移行機能であり、オフラインとオンラインの両方の移行方式をサポートしています。ONTAPでは専門サービス主導の移行は必須ではありませんが、特に複雑な環境においては、範囲設定、計画策定、およびトレーニングに関してNetAppまたは資格のあるパートナーを活用することを強くお勧めします。



== 適切な移行方法の選択（概要）

実践的な移行戦略では、主要な方法を一つ採用し、状況に応じて他の手法を補完的に用いることが考えられます。SAN/LUNレベルのONTAPへの移行の場合、FLIは一般的に有力な選択肢ですが、状況によっては、設計上無停止で実行できる代替手段の方が適している場合もあります。

.共通オプション：
Foreign LUN Import（FLI）:: オンラインおよびオフラインのワークフローを使用した、ONTAPへのブロック/LUNレベルの移行。
Data Transfer Appliance（DTA）:: SANに接続されたアプライアンスモデルで、移行したデータ量（TB単位）に応じてライセンスが付与され、オフラインおよびオンライン移行をサポートします。
ホストOSまたはアプリケーションベースの移行:: プラットフォームがネイティブなモビリティ機能を提供する場合、またはホストやアプリケーションの制御が優先される場合に役立ちます（例：VMware のストレージモビリティやホストベースのコピー方式）。




== 移行期間の計画

初期計画段階に限っては、一般的に引用されるベンチマークの想定として、合計約2TBの8つのLUNを持つホストの場合、ホスト1台あたり約5時間、つまり約400GB/時間というものがあります。これは意図的に大まかな計画値として算出されたものであり、あらゆる環境に当てはまるとは限りません。正確な見積もりを得るための推奨方法は、さまざまな規模のテスト移行を複数回実行し、自社のストレージ、ネットワーク、ホスト環境における実際のスループットを測定することです。



== 推奨ツール（データ収集、互換性、自動化、評価）

サービスツールは、環境データを収集し、サポート可能性を検証し、計画および実行段階における人的ミスを削減するための標準化された方法を提供します。

Active IQ OneCollect:: トラブルシューティング、検証、移行計画、およびアップグレード評価のために、ストレージ、ホスト、ファブリック、およびスイッチからデータを収集します。
Interoperability Matrix Tool（IMT）:: NetAppおよびサードパーティ製コンポーネント全体でサポートされている構成を確認するためのWebベースのユーティリティ。
Solaris Relabeler（該当する場合）:: ブロックレベルの移行後、システムが更新された照会情報を反映するように、移行後にVTOCディスクラベルを更新します。
SAN LUN Migrate（FLIプロジェクト用のsanlunmigrate）:: 本番環境への移行に着手する前に、顧客環境における外部アレイ／LUNの動作を検証するための認定アプローチを使用してください。これは、トランスポートやベンダーの動作が異なる場合に特に重要です。

= ONTAP Foreign LUN Importでサポートされている構成とプラットフォームについて学ぶ
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
ONTAP FLIは、予測可能な動作と完全なサポートを保証するために、サポートされている構成で展開する必要があります。

* すべてのFLI移行はLUN（ブロック）レベルで実行されます。ファイルおよびプロトコルレベルの移行は、ONTAP FLIの範囲外です。
* ノードごとに最大8件の同時インポートを想定してください。実際に安全に実行できる同時実行数は、観測されるホストの遅延時間とインポートの遅延時間に依存します。
* インポート可能な外部LUNの最大サイズは、LUNあたり4TBです。
* iSCSIセッションのスケーリング制限（バックエンドiSCSI）：
+
** IC LIFあたり1つのイニシエーター
** LIFあたり最大16セッション
** ノードあたり最大32セッション


* iSCSIプロトコルの制約事項：
+
** iSCSI検出セッションなし
** マルチ接続セッション（MCS）なし
** エラー回復レベル > 0 はサポートされていません（ERL0 のみ）
** iSCSIイニシエータLIFの使用ではIPv6はサポートされていません


* セキュリティおよび管理の可用性に関する注意事項：
+
** CHAPは含まれていません
** iSCSI FLI操作のREST APIカバレッジは利用できません（CLIワークフローのみ）。


* オンライン インポートに関する運用計画メモ：
+
** オンラインFLIでは、ホストI/Oとインポートコピーがバックエンドリソースを共有します。
** ホストのレイテンシを保護するために、インポートを一時停止および再開できます
** 可能であれば、移行はピーク時以外の時間帯にスケジュールする必要があります






[[supported-configurations-and-platform]]
== サポートされている構成とプラットフォーム

ONTAP FLIは、予測可能な動作と完全なサポートを保証するために、サポートされている構成で展開する必要があります。時間の経過とともに新しい構成が認定されるにつれて、サポートされる組み合わせが変わる可能性があります。各移行について、エンドツーエンドのスタック（ソースアレイ+ホスト+ファブリック/ネットワーク+ONTAPターゲット）を現在の相互運用性に関するガイドラインに照らして検証してください。

.サポートされている移行先（移行可能な場所）
* ONTAPは、FLI移行においてサポートされている唯一の移行先です。FLIは、データをONTAP LUNにインポートするように設計されています。サードパーティの移行先ストレージへの移行はサポートされていません。
* iSCSI FLIの場合、iSCSIソフトウェアイニシエータはFLIスタックに統合されており、iSCSI/TCPバックエンド接続を使用して外部LUNの検出とインポートを可能にすることを目的としています。


.サポート対象ONTAPプラットフォームカバレッジ（デスティネーションプラットフォーム）
プラットフォームの対応範囲は、検出/インポートに使用されるバックエンドトランスポートによって異なります：

* FC バックエンド FLI（従来型）は、AFF / ASA / ASA r2 / FAS 上の ONTAP をサポートします（iSCSI テスト設計の比較表を参照）。
* iSCSIバックエンドFLIは、AFF / ASA / ASA r2 / FAS / ONTAP Select / Cloud Volumes ONTAP（CVO）まで適用範囲を拡大します（同じ比較による）。


.サポートされているソースアレイと認定方法
* FCバックエンドFLIでは、サポートされる外部アレイは通常、確立された相互運用性リストと認定ガイダンスを通じて検証されます（サポートされるリストは、新しいアレイが認定されるにつれて更新されます）。
* iSCSIバックエンドFLIでは、デフォルトでは外部アレイの事前認定は行われません。お客様は、本番環境への移行を実行する前に、提供されている認定ツール（例：sanlunmigrate / offtap sanlunmigrate）を使用して現場での認定を実施する必要があります。


.ホストオペレーティングシステムおよびファブリック／スイッチのサポート（エンドツーエンドの検証）
* ホストOSのサポートは、選択したフロントエンドプロトコル（FCまたはiSCSI）および移行後のホスト構成に関するSANホスト相互運用性ガイドラインに準拠する必要があります。
* スイッチ／ファブリックのサポートも、エンドツーエンド構成の一部として検証する必要があります（特に、ゾーニング／マスキングが必要なFC環境の場合）。


.移行後の要件（カットオーバー後もサポートを継続）
インポートが完了し、ホストがONTAP LUNで稼働したら、最終的な安定状態構成がサポートされていることを確認してください。これには、ホストのマルチパス、HBA/ドライバ/ファームウェアレベル、ゾーニング/マスキング（FC）、iSCSI構成（使用する場合）、および使用予定のONTAP機能の相互作用が含まれます。

.iSCSI FLI固有のガードレール（サポート性にとって重要）
* iSCSI FLIは、バックエンドiSCSIセッションにインタークラスタ（IC）LIFを使用します。別のサブネット上に既存のIC LIFがすでに確立されている場合に、外部ターゲットに到達するためだけに別のサブネット上に新しいIC LIFを作成することは、サポートされていない計画ガイダンスとして扱われます。
* iSCSIイニシエータセッションの設定は、移行のためだけに存在し、アップグレードやリバートなどの運用上の副作用を避けるため、移行完了後にはクリーンアップする必要があります。




[[best-practices]]
== ベストプラクティス

移行の範囲設定や計画の支援のほか、お客様の担当者向けの Foreign LUN Import （ FLI ）を使用した 7-Mode から ONTAP へのデータ移行のトレーニングも受けられるため、プロフェッショナルサービスまたはパートナープロフェッショナルサービスを利用することを強く推奨します。

* 移行プロジェクトの少なくとも1週間前に、1回以上のテスト移行を実施して、構成、接続性、スループットを確認し、その他の問題点を明らかにし、方法論の妥当性を検証してください。
* スループットが最大になるように、各ノードで同時に実行する移行の数は 16 個までにします。
* 検証は必須ではありませんが、インポートおよび移行されたLUNの一部を検証して、インポートプロセスを検証することをお勧めします。
* テスト移行で観測されたスループットを使用して、本番環境への移行を計画します

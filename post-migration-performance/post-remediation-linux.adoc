= ONTAP FLI Linux移行後のホスト修復を実行する
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
移行後、Linuxホストでは、新しくインポートされたLUNが正しく認識され、正常に機能するようにするための修復手順が必要です。



[[post-migration-offline-fli]]
== 移行後 - オフライン FLI

オフラインFLIの場合は、LUNインポートを削除し、ホストイニシエータが接続されていることを確認した後、LUNをオンラインにしてください。

[cols="3"]
|===
| ファイルシステム（LVMおよび非LVM） | LVM搭載Pacemaker（SCSI3 PR） | SANBOOT 


| ステップ1：マッピングされたすべてのLinuxホストでストレージを再スキャンします | ステップ1：マッピングされたすべてのLinuxホストでストレージを再スキャンします | ステップ1：サーバーの電源を入れる 


| ステップ2：マルチパスデバイスを検出する。 | ステップ2：マルチパスデバイスを検出する。移行したLUNがPCS Stonith構成に追加されていることを確認してください。 | ステップ2：サーバーのHBA BIOSまたはUEFIで、このホストHBAに提示される新しいブートデバイスを設定します 


| ステップ3：ボリュームグループをインポートし、VGのアクティベーションを実行します。 | ステップ3：新しく検出されたVG/LVMに対してVG構成バックアップの復元を実行し、その後VGをアクティブ化します。 | ステップ3：ホストを再起動し、LUN Importの一部である新しくマッピングされたLUNから起動します。 


| ステップ4：LVM構成をそれぞれのマウントポイントにマウントします。 | ステップ4：新しいPCSリソースグループを作成し、利用可能なLVとファイルシステムに対応するリソースを追加します。注：新しくアクティブ化されたLVMオブジェクトは、LVおよびファイルシステム構造をそのまま保持していることにご注意ください。 |  


|  | ステップ5：すべてのリソースがオンラインで利用可能なステータスであることを、PCSクラスタのステータスで確認します。 |  
|===


[[post-migration-online-fli]]
== 移行後 - オンラインFLI

LUN インポートを削除した後、LUN が `-space-reserve disabled`および `-space-allocation enabled`で設定されているかどうかを確認します。そうでない場合は、それに応じて修正してください。

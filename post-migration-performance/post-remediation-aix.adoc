= ONTAP FLI AIX移行後のホスト修復を実行する
:allow-uri-read:

[role="lead"]
移行後、AIXホストでは、新しくインポートされたLUNが正しく認識され、正常に機能するようにするための修復手順が必要です。



[[post-migration-offline-fli]]
== 移行後 - オフライン FLI

オフラインFLIの場合は、LUNインポートを削除し、ホストイニシエータが接続されていることを確認した後、LUNをオンラインにしてください。

| ステップ2：マルチパスデバイスを特定する | ステップ2：マルチパスデバイスを特定する | ステップ2：マルチパスデバイスを特定する | ステップ2：マルチパスデバイスを特定する | ステップ2：IPL（Initial Program Load）/SMSメニューからブートデバイスリストを確認すると、新しいStorageデバイスから取得したデバイスリストを参照する新しいLUNが表示されます。ブート順序を新しいデバイスリストパスに設定します

NOTE: 最新および更新されたコマンドリストについては、OSベンダーのドキュメントを参照してください。

.手順
. 取り外し後に再度スキャンして、クリーンな状態であることを確認してください。
+
[source, cli]
----
cfgmgr
----
. ストレージデバイスを特定します。
+
[source, cli]
----
lsdev -Cc disk
sanlun lun show all
sanlun lun show all -pv
lsmpio -l hdiskx
----
. デバイスが接続されていることを確認してください。
+
[source, cli]
----
lspv |grep hdiskx
----
. ボリュームグループとファイルシステムを特定します。
+
[source, cli]
----
lspv
lsvg <vg_name>
lsvg -l <vg_name>
lsvg -p <vs_name>
----
. Varyonvgとマウントファイルシステム。
+
[source, cli]
----
varyonvg <vg_name>
mount /mount_point
df -g
----




[[post-migration-online-fli]]
== 移行後 - オンラインFLI

インポート関係を削除した後、新しいLUNがUNMAPを有効にした状態で独立して動作することを確認してください。LUNインポートを削除した後、LUNが `-space-reserve disabled`および `-space-allocation enabled`で設定されているかどうかを確認します。設定されていない場合は、それに応じて修正してください。

[cols="4"]
|===
| Filesystem | ブロックデバイス | LVM（FS） | Sanboot 


| ステップ1：マッピングされたすべてのAIXホストでストレージを再スキャンします | ステップ1：マッピングされたすべてのAIXホストでストレージを再スキャンします。 | ステップ1：マッピングされたすべてのAIXホストでストレージを再スキャンします | ステップ1：マッピングされたすべてのAIXホストでストレージを再スキャンします 


|  |  | デバイスがシンプロビジョニングされている場合、SCSI UNMAPは自動的に有効になります。 | デバイスがシンプロビジョニングされている場合、SCSI UNMAPは自動的に有効になります。 
|===
.手順
. 取り外し後に再度スキャンして、クリーンな状態であることを確認してください。
+
[source, cli]
----
cfgmgr
----
. ストレージデバイスを特定します。
+
[source, cli]
----
lsdev -Cc disk
sanlun lun show all
sanlun lun show all -pv
lsmpio -l hdiskx
----
. デバイスが接続されていることを確認してください。
+
[source, cli]
----
lspv |grep hdiskx
----
. デバイス上でUNMAPを確認する：
+
[source, cli]
----
chfs -a reclaim=normal /mount_point
lvmstat -v vg_name -r
----

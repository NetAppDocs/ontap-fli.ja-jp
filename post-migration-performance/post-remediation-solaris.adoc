= ONTAP FLI Solaris 移行後のホスト修復を実行
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
移行後、Solarisホストでは、新しくインポートされたLUNが正しく認識され、正常に機能するようにするための修復手順が必要です。



[[post-migration-offline-fli]]
== 移行後 - オフライン FLI

オフラインFLIの場合は、LUNインポートを削除し、ホストイニシエータが接続されていることを確認した後、LUNをオンラインにしてください。

[cols="5"]
|===
| Filesystem | ブロックデバイス | LVM（FS） | クラスタ | Sanboot 


| ステップ1：マッピングされたすべてのSolarisホストでストレージを再スキャンします | ステップ1：マッピングされたすべてのSolarisホストでストレージを再スキャンします。 | ステップ1：マッピングされたすべてのSolarisホストでストレージを再スキャンします | ステップ1：マッピングされたすべてのSolarisホストでストレージを再スキャンします | ステップ1：サーバーの電源を入れる 


| ステップ2：マルチパスデバイスを特定する | ステップ2：マルチパスデバイスを特定する | ステップ2：マルチパスデバイスを特定する | ステップ2：マルチパスデバイスを特定する | ステップ2：SPARCサーバーのOBP上で、このホストHBAに提示される新しいブートデバイスを設定します。 


| ステップ3：プールの詳細を確認する | ステップ3：マルチパスデバイスIDを特定し、それを該当するアプリケーションに提示します。 | ステップ3：プールのステータスを確認する | ステップ3：ファイルシステムの詳細を確認する | ステップ3：ホストを再起動し、LUN Importの一部である新しくマッピングされたLUNから起動します。 


| ステップ4：デバイスをそれぞれのフォルダに再マウントする |  | ステップ4：プールをインポートする | ステップ4：必要なクラスタ構成を確認し、クラスタアプリケーションを起動します |  


|  |  | ステップ5：プールに存在するFSをリストアップする | ステップ 5：クラスタのステータスと消費しているアプリケーションを検証します。 |  


|  |  | ステップ6：FSをそれぞれのフォルダにマウントし直す |  |  
|===

NOTE: 最新および更新されたコマンドリストについては、OSベンダーのドキュメントを参照してください。

.手順
. 接続後に再スキャンします。
+
[source, cli]
----
cfgadm -al
----
. ストレージデバイスを特定します。
+
[source, cli]
----
format
sanlun lun show all
mpathadm list lu
mpathadm show lu
----
. デバイスが接続されていることを確認してください。
+
[source, cli]
----
format |grep disk_id
----
. zpoolをインポートすると、FSが自動的にマウントされます。
+
[source, cli]
----
zpool import zpool_name
df -g (to ensure its unmounted)
----

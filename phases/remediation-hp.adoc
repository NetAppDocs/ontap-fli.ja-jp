= ONTAP FLI HPホストの修復
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
HPホストでONTAP FLI移行を実行する前に、以下の移行前手順を完了してください。オフライン移行とオンライン移行には、それぞれ別の手順が必要です。



[[pre-migration-offline-fli]]
== 移行前 - オフラインFLI

これらの手順は、オンラインFLIとオフラインFLIの両方に共通です。ホストからソースLUNのプロビジョニングを解除する前に、以下の手順が完了していることを確認してください。

[cols="3"]
|===
| ファイルシステム（LVMおよび非LVM） | ホストクラスタ（SCSI3 PR） | SANBOOT 


| ステップ1：エンドツーエンドのブロックデバイス構成を収集する：マルチパスの健全性/パス、ブートLUN上のLVMレイアウト、および正しいブートデバイスとブートパラメータを確認するUEFI設定。 | ステップ1：クラスタ全体の詳細情報と、移行に関係するディスクを取得します。 | ステップ1：エンドツーエンドSANブート構成を収集する：マルチパスの健全性/パス、ブートLUN上のLVMレイアウト、および正しいブートデバイスとブートパラメータを確認するUEFI設定。 


| ステップ2：それぞれのデータデバイス（LVM/非LVM）の/etc/fstabエントリが適切であることを確認します。 | ステップ2：推奨手順に従って、クラスタサービスを正常にシャットダウンします。 | ステップ2：データデバイス（lvm/nonlvm）を含むSANブートデバイスの/etc/fstabエントリが適切であることを確認します 


| ステップ3：ターゲットマウントポイントへのすべての操作が一時停止されていることを確認します。ブロックデバイスにアクセスしているプロセスが残っていないことを確認してください。必要に応じて、開いているプロセスハンドルをドレイン/強制終了します。 | ステップ3：マルチパスデバイスに関連付けられているSCSIブロックデバイスを削除します。 | ステップ3：UEFIシェルからデバイスマッピングテーブルを確認し、ストレージデバイスからhpuxを起動します 


| ステップ4：ファイルシステムをアンマウントし、LVMボリュームグループ構成のバックアップを実行します。VGを非アクティブ化してVGをエクスポートする | ステップ4：ホストを再スキャンして、既存のデバイスマッピングに古いエントリがないことを確認します。 | ステップ4：プライマリブートデバイスがStorage deviceに設定されていることを確認してください。これにより、以降の再起動時に自動的に起動するようになります。 


| ステップ5：古いSCSIデバイスファイルをすべて削除します。 |  | ステップ5：ホストをリブートして、上記の変更でホストが正常に起動できることを確認します。ホストが正常に起動したら、FLI移行のためのホスト事前修復の準備が整ったことを確認します。 


| ステップ6：ホストが最新のデバイスリストを取得できるように、sciバスを再スキャンします（例：ioscan -fnC disk） |  | ステップ 6：次に sanboot ホストをシャットダウンして、ONTAP FLI ワークフローを続行します。 
|===


[[pre-migration-online-fli]]
== 移行前 - オンラインFLI

以下はオンライン FLI の具体的な手順ですが、これらの手順の前には移行前オフライン FLI の手順を実行する必要があります。以下の手順は、NetApp LUN がホストにプロビジョニングされた後に実行する必要があります。

[cols="4"]
|===
| Filesystem | ブロックデバイス | LVM（FS） | Sanboot 


| ステップ1：マッピングされたすべてのHP-UXホストでストレージを再スキャンします | ステップ1：マッピングされたすべてのHP-UXホストでストレージを再スキャンします。 | ステップ1：マッピングされたすべてのHP-UXホストでストレージを再スキャンします | ステップ1：サーバーの電源を入れる 


| ステップ2：マルチパスデバイスを特定する | ステップ2：マルチパスデバイスを特定する | ステップ2：マルチパスデバイスを特定する | ステップ2：UEFIシェルからデバイスマッピングテーブルを確認し、新しいストレージデバイスからhpuxを起動します 


| ステップ3：ファイルシステムの詳細を確認する | ステップ3：マルチパスデバイスIDを特定し、それを該当するアプリケーションに提示します。 | ステップ3：新しいLVMデバイスファイルが正しく作成されていることを確認します | ステップ3：プライマリブートデバイスを新しいStorage LUNに変更します。ホストを再起動して、LUN Importの一部である新しくマッピングされたLUNから起動します。 


| ステップ4：デバイスをそれぞれのフォルダに再マウントする |  | ステップ4：VG/LVMをインポートする |  


|  |  | ステップ5：VGをアクティブ化する |  


|  |  | ステップ6：VG上に存在するLVをリストアップする |  


|  |  | ステップ7：LVMをそれぞれのフォルダに再マウントする |  
|===

NOTE: 最新および更新されたコマンドリストについては、OSベンダーのドキュメントを参照してください。

.手順
. ストレージデバイスを特定します。
+
[Source]
----
ioscan -fnC disk
sanlun lun show all (Works on Netapp Storage)
sanlun lun show all -pv
sanlun lun show -wwpn
----
. ボリュームグループとファイルシステムを特定します。
+
[Source]
----
vgdisplay -v /dev/vg_name
vgdisplay /dev/vg_name
lvdisplay /dev/vg_name/lv_name
----
. ファイルシステムをアンマウントし、VGをエクスポートします。
+
[Source]
----
bdf
umount /mount_point
vgchange -a n /dev/vg_name
vgexport /dev/hpux_vg_online
----
. 取り外し後に再度スキャンして、クリーンな状態であることを確認してください。
+
[Source]
----
insf -e
ioscan -fnC disk
----

= ONTAP外部LUNインポート（FLI）検証フェーズ
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
検証フェーズでは、移行が計画通りに完了したことを確認し、移行後のクリーンアップ作業に重点を置きます。これには、アプリケーションの動作の検証、ONTAP LUNへの正しいホストアクセスとパスの確認、監査およびトラブルシューティングのための移行ログの収集、移行のためだけに使用された一時的な接続および構成の削除が含まれます。

クリーンアップの一環として、不要になったソース側のイニシエータレコードとゾーニングを削除し、運用チームが明確な「構築済み」状態を把握できるよう、最終的な構成を文書化します。

iSCSI FLIの場合、検証フェーズには、インタークラスタLIF上に作成されたソフトウェアiSCSIイニシエータセッションおよびイニシエータの削除も含まれるべきです。インタークラスタLIFが外部LUNのインポートのためだけに作成された場合、それらのLIFも削除する必要があります。



[[ontap-fli-verify-phase-tasks]]
== ONTAP FLI検証フェーズのタスク

以下は、各コンポーネントのONTAP FLI検証タスクです。

[cols="1,2"]
|===
| コンポーネント | タスク 


| ホスト  a| 
* 移行されたLUNに対して、アプリケーションレベルの検証を実行します。（ファイルシステムの一貫性チェック、チェックサムエラー、dbcc checkdb (MS SQL)、rman (Oracle)）
* 移行および修復後、想定されるすべてのファイルシステム、データストア、クラスタディスクにアクセス可能で安定していることを確認してください。
* IOパス数とパスタイプが想定される行に表示されていることを確認してください。




| FC FLIファブリック  a| 
* 初期カットオーバー時に削除されていない場合は、アクティブゾーンセットからホストとソースストレージ間のゾーン、または同等のアクセスルールを削除します。
* 移行専用に作成された、送信元と送信先間の接続ゾーンを削除します。
* ロールバックおよび監査のために、更新後の移行ゾーンセットをバックアップします。




| iSCSI FLIネットワーク  a| 
* 移行用に作成されたiSCSI FLIバックエンド接続オブジェクトを削除します。
* iSCSIイニシエータセッションを削除し、クラスタ間LIF上に作成されたiSCSIイニシエータオブジェクトを削除する必要があります。




| NetAppストレージアレイ（宛先）  a| 
* 移行後のストレージの正確性を検証します。ホストまたはユースケースで必要な場合は、LUNアライメントも確認してください。
* 監査およびトラブルシューティングのために、FLI移行インポートログを収集してください。移行が完了したら、AutoSupportをトリガーします。
* 移行期間中のロールバックのみを目的として手動で作成されたSnapshotコピーはすべて削除してください。
* ソースアレイオブジェクトや外部ディスク関連付けの詳細など、不要になったONTAPのソースアレイ構成アーティファクトを削除します。storage disk showコマンドを使用して確認します。
* LUNパス、マッピング、igroup、イニシエータ、ホストアクセスモデルなど、最終的なONTAP構成を文書化します。




| 外部ストレージアレイ（ソース）  a| 
* 移行中にソースLUNをONTAPに提示するために作成された宛先ストレージ（NetApp）イニシエータグループとホストグループを削除します。
* ソースアクセスを廃止する際は、元のホストマッピングに関連するホストイニシエータグループとホストレコードを削除してください。
* 移行されたソースLUNが、ホストに対して二重提示を引き起こす可能性のある方法で提示されていないことを確認してください。これは非常に重要であり、移行全体を通して確実に実施する必要があります。


|===

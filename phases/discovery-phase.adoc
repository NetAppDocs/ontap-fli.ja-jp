= ONTAP Foreign LUN Import（FLI）検出フェーズ
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
ONTAP FLI検出フェーズは、移行プロセスの最初のステップです。ソース環境、接続性、ターゲットONTAPシステムに関する必要な情報をすべて収集することが、スムーズな移行を実現するために極めて重要です。検出フェーズは、移行の計画、準備、実行、および検証の基礎を築きます。



[[prepare-for-an-iscsi-migration-with-ontap-fli]]
== ONTAP FLIを使用したiSCSI移行の準備

ONTAP FLI検出フェーズでは、移行を完了するために必要な技術的な詳細情報を収集します。

* 移行ワークフローを計画する（オンラインかオフラインか）
* ホスト修復を安全に実行する
* インポート関係を作成する前に、ONTAPで外部LUNの接続性と検出を検証します


ホスト、NetAppコントローラ、ファブリック全体で構造化された反復可能なデータ収集を行うには、NetApp Active IQ OneCollectを使用できます。OneCollectは、移行の計画と検証のために、ストレージ、ホスト、ファブリック、スイッチから情報を収集するように設計されています。

FC FLIの場合、検出フェーズでは、外部アレイのターゲットポートのWWPNを取得する必要があります。

iSCSI FLIの場合、検出フェーズでは次の操作を実行する必要があります：

* 外部アレイのターゲットポータルIPとターゲットIQNをキャプチャします
* IC LIFの到達可能性を確認する
* 外部LUNが利用可能なすべてのパスを通じて検出されるように、フルメッシュiSCSIセッション（各HAノードが各ターゲットポータルに接続する）を計画してください




[[ontap-fli-discovery-phase-tasks]]
== ONTAP FLI検出フェーズのタスク

各コンポーネントには、データを収集したり接続を検証したりするための一連のタスクがあります。

[cols="1,2"]
|===
| コンポーネント | タスク 


| ホスト  a| 
* ホストOSのバージョン、マルチパススタックと構成、およびファイルシステム、RAWデバイス、クラスタディスクなどの特別な消費モデルを特定し、ホストの修復および切り替え手順を推進します。
* LUNのサイズ、識別子またはシリアル番号、パーティションレイアウトまたはオフセット、および各LUNがアプリケーションによってどのように使用されているかなど、LUNのインベントリ情報を取得し、計画およびロールバックに役立てます。
* FCフロントエンド環境の場合、ホストイニシエータのWWPNを記録します。これらは、全体的な移行および切り替え設計に該当する場合、ゾーニングおよびマスキングに使用されます。
* iSCSIフロントエンド環境の場合、移行後に使用する場合は、カットオーバー後にホストがログインに使用するホストイニシエータIQNとターゲットポータルIPを記録します。




| ファブリックとネットワーク  a| 
* FC環境の場合：
+
** 切り替え後に安全に削除または更新できるよう、現在のゾーニング（ホストおよびソースアレイゾーンとの間のゾーニング）を文書化してください。
** 関係するスイッチ識別子とポートをキャプチャします。


* iSCSI環境の場合：
+
** ONTAP IC LIFと外部アレイターゲットポータル間のVLAN、サブネット、ルーティングパスを文書化する
** 通信をブロックするファイアウォールやACLがないことを確認してください。






| NetAppストレージアレイ（デスティネーション）  a| 
* iSCSI FLI の ONTAP バージョンと ECV の準備状況を確認します。
+

NOTE: クラスタがサポートされているレベルでない場合、イニシエータの作成はブロックされます。

* LUNを所有するノードとそのHAパートナーなど、どのノードが参加するかを特定し、それらのノードで使用可能なクラスタ間LIFを記録します。冗長性を考慮し、両方のHAノードでIC LIF、イニシエータ、およびセッションを計画してください。
* IC LIFのIPspaceとサブネットを記録し、それらが外部のターゲットポータルに到達できることを確認します。クラスタ内の他の場所で既存のIC LIFがすでに使用されている場合は、別のサブネットに新しいIC LIFを作成しないでください。
* 計画段階で適用する必要のある宛先側の制約（インポートの最大サイズ、512Bのブロックサイズ要件、iSCSI FLIに関連するセッションおよびインポートのスケーリング制限など）を検証します。




| 外部ストレージアレイ（ソース）  a| 
* 外部アレイのターゲットポータルIPアドレスとターゲットIQNを特定します。iSCSI FLIではiSCSI検出セッションはサポートされていないため、これらは事前に明示的に把握しておく必要があります。
* ソースホストにマッピングされている特定のLUNを特定します。LUNサイズとブロックサイズも併せて示してください。LUNが安定しており、拡張されておらず、インポート計画に適していることを確認してください。
* ホストレコードやイニシエータレコードなどのホストアクセスおよびマスキングオブジェクトを記録しておくことで、切り替え時に安全にマッピングを解除したり、再マッピングしたり、ロールバックの準備状況を検証したりすることができます。
* 外部アレイの適格性評価を計画し、実施する。
+

NOTE: iSCSI FLIには現場での認定が必要です。FC認定済みのアレイが自動的にiSCSI認定済みであるとは想定しないでください。



|===


[[recommended-ontap-fli-discovery-deliverables]]
== 推奨されるONTAP FLI検出成果物

* 単一のインベントリシートのマッピング：ホスト → ソースLUN（複数可） → 外部ターゲットポータル/IQN → ONTAP IC LIF/ノード → クリーンな実行とロールバックのための宛先LUN（複数可）。
* 変更実行前にキャプチャされたOneCollectパッケージ（または同等のもの）により、問題が発生した場合のトリアージ時間を短縮できます。

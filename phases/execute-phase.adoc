= ONTAP Foreign LUN Import（FLI）実行フェーズ
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
実行フェーズは、移行が実行されるフェーズです。オフラインまたはオンラインのForeign LUN Importを実行し、ホストアクセスを外部アレイからONTAPアレイに移行するために必要なエンドツーエンドのタスクをカバーします。



[[ontap-fli-execute-phase-tasks]]
== ONTAP FLI実行フェーズのタスク

以下は、各コンポーネントに対するONTAP FLI実行タスクです。

[cols="1,2"]
|===
| コンポーネント | タスク 


| ホスト  a| 
* 大規模な再構成を行う前に、ホストのイベントログとシステムログを確認して問題を早期に発見し、リスクを軽減してください。
* 早期カットオーバーまたは修復の前に、安定性を確保するために必要に応じてホストを再起動します。
* 切り替え後、ホストアクセスを検証します。想定されるすべてのデバイスが存在し、マルチパスが正常で、アプリケーションとファイルシステムが正常に起動します。




| iSCSI FLIとネットワーク  a| 
* IC LIFが外部アレイのターゲットポータルに到達可能であることを確認し、インポート全体を通してバックエンドの接続が安定していることを確認してください。
* iSCSIセッションが正常な状態を維持していることを確認してください。iSCSIセッションは、外部LUNの検出とパスの可用性を左右するからです。




| ファブリック  a| 
* ONTAPイニシエータポートが検出とインポートのために外部アレイLUNにアクセスできるように、必要なゾーニングとマスキングが設定されていることを確認してください。
* 切り替え後、必要に応じてゾーニングを更新して、ホストがデスティネーションONTAP LUNにアクセスできるようにし、後でクリーンアップできるように準備します。




| NetAppストレージアレイ（宛先）  a| 
* インポートジョブを作成する前に、ONTAPで外部LUNが検出されていることを確認します。
* 検出が確認されたら、インポート関係を作成し、インポートジョブを開始します。
* インポートの進行状況を監視し、例外を処理します。必要に応じて一時停止と再開を使用してコピー ストリームを調整してください。特にホストのレイテンシが増加する場合は有効です。
* 切り替え処理：
+
** オフラインFLIカットオーバーは、インポート完了後に行われます。
** オンラインFLIホストのカットオーバーは、インポートがバックグラウンドで継続されている間に、より早く行われます。






| 外部ストレージアレイ（ソース）  a| 
* 移行対象のソースLUNが、ONTAPによる検出とインポートのために正しく提示されていることを確認してください（トランスポートに応じて、FCマスキングまたはiSCSIターゲットの可視性）。
* インポート中は、ホスト経由でソースLUNにアクセスしないようにしてください。これは、書き込み操作が複数のソースから発生しないようにするために非常に重要です。
* オンライン切り替え中は、二重表示を防ぎ、ホストがスムーズにONTAPに移行できるように、必要に応じてマッピング解除とマッピングの再設定を調整します。


|===
FCとiSCSIバックエンドFLIの両方において、実行の成功は、FC/iSCSIセッションの健全性を確保し、外部LUNへの完全パス冗長性を維持することにより、安定したバックエンド接続を維持することに大きく依存します。ホストのI/O（オンライン）とインポートコピーは同じバックエンドリソースを競合する可能性があるため、ホストのレイテンシを保護するために必要に応じてインポートが運用的に制限される場合があり（例えば、一時停止と再開を使用するなど）、移行は通常、ピーク時以外の時間帯にスケジュールされます。

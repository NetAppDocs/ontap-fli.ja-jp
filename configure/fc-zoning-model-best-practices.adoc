= ONTAP Foreign LUNインポートにおける推奨FC配線およびゾーニング方法
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
外部LUNインポート（FLI）では、ソースLUNがONTAPからアクセス可能である必要があります。これは、ソースターゲットポートをONTAPイニシエーターポートにゾーニングすることによって実現されます。既存のソースとホスト間のゾーンは通常、移行中に変更されることはなく、移行後に削除または無効化することができます。

.標準的なFC FLIシナリオでは、4つの異なるゾーンを使用します
* ゾーン1：ソースストレージから宛先ストレージへ（ファブリックA）
* ゾーン2：ソース ストレージから宛先ストレージへ（ファブリックB）
* ゾーン3：ホストから宛先ストレージへ（ファブリックA）
* ゾーン4：ホストから宛先ストレージ（ファブリックB）


.ゾーニングのベストプラクティス
* 送信元ターゲット ポートと宛先ターゲット ポートを同じゾーンに混在させないでください。
* 宛先イニシエータ ポートとホスト ポートを同じゾーンに混在させないでください。
* 宛先ターゲット ポートと宛先イニシエータ ポートを同じゾーンに混在させないでください。
* 可能な限り、冗長性を確保するため、コントローラごとに最低2つのポートを備えたゾーン。
* 単一のイニシエータと単一のターゲットによるゾーニングを優先する。
* FLIワークフローで必要とされるゾーニングモデルに適した物理ポートのWWPNを使用してください。移行ゾーニングガイダンスで明示的に禁止されている場合は、NPIV WWPNを使用しないでください。

NOTE: ソース ターゲットを宛先イニシエータにゾーニングした後、ソース ストレージはONTAPから検出コマンドを介して認識されるようになります。アレイが最初に表示されない場合は、環境によってはONTAPイニシエータに接続されているスイッチ ポートをリセットする必要があります。

= ONTAP FLI構成について学ぶ
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
このセクションでは、ファイバチャネル（FC）とiSCSIバックエンド接続の両方を使用した外部LUNインポート（FLI）の構成および検証フレームワークについて説明します。LUNの検出およびインポート中に、ONTAPから外部ストレージアレイへの信頼性が高く冗長性のあるアクセスを確保しつつ、ホストアクセスパスとは明確に分離した状態を維持することに重点を置いています。このコンテンツでは、推奨されるSAN設計原則（デュアルファブリックFC、イニシエータ/ターゲットポートの使用、ゾーニングのベストプラクティス）、ONTAPのアダプタ構成要件、およびインポートを開始する前に外部LUNが認識可能で正常であることを確認するための運用チェックについて説明します。

さらに、iSCSIバックエンドFLIモデルの詳細も説明されており、ONTAPソフトウェアiSCSIイニシエータは、インタークラスタLIF（IC LIF）にバインドされ、明示的なセッション作成（検出セッションなし）、回復力のためのフルメッシュ接続、およびネットワーク到達可能性とセッション健全性の厳密な検証を重視します。FCとiSCSIの両方のシナリオにおいて、目標は、FLIの前提条件を満たし、すべてのパスで一貫した外部LUNの検出を可能にし、検出およびインポート実行中の問題のトラブルシューティングに必要な計測機能（統計情報と可視性）を提供する、決定論的でサポート可能なバックエンドトランスポートを確立することです。

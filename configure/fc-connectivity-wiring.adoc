= ONTAP外部LUNインポート（FLI）接続と配線
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
FC を介した外部 LUN インポートは、デュアルファブリック SAN 設計に依存しているため、宛先 ONTAP コントローラは、検出およびインポート中に外部アレイ LUN に確実にアクセスできます。サポートされているデプロイメントの場合、使用されている各ストレージ コントローラは、両方のファブリックにプライマリ アクティブ パスを持っている必要があります。これは実質的に、ソース アレイ ターゲット ポートと、インポートを実行するノードの ONTAP イニシエータ ポートは、ファブリック A とファブリック B の共通ゾーンにある必要があります。ONTAP クラスター内のすべてのノードを含める必要はありません。実際に LUN をインポートおよび移行するノードのみです。間接的なパスでも機能することはありますが、ベスト プラクティスは、ソースとデスティネーションの間でアクティブで最適化されたパスを使用することです。



[[ontap-fli-recommended-fc-wiring-and-zoning-practices]]
== ONTAP FLI推奨FCケーブル接続およびゾーニングのプラクティス

* 冗長性を確保するため、デュアル ファブリック（AとB）を使用してください。
* 使用中の各コントローラからのプライマリのアクティブ パスが両方のファブリックに存在することを確認します。
* 単一障害点を回避するため、可能な限り少なくとも2つのONTAPイニシエータポートと2つのソースアレイターゲットポートを使用してください。
* ONTAPイニシエータポートをホストとゾーン分けしないでください。ONTAPイニシエータポートは、インポートおよび検出のためにソースアレイのターゲットポートに接続するためにのみ使用されます。
* インポートに参加するONTAPノードのみをゾーニングします。クラスタ全体をゾーニングする必要はありません。
